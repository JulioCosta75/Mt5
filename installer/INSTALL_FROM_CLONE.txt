Atlas - Install from a fresh git clone (Windows)
================================================

This is the quickest way to get Atlas running on a Windows machine
(or VPS) straight from the repository, without building the full
Atlas_Setup.exe installer.

Prerequisites
-------------
  * Windows 10/11 or Windows Server (x64)
  * Python 3.10 - 3.12  (tick "Add python.exe to PATH" during install)
  * Node.js + Yarn      (only needed to build the dashboard)
  * MetaTrader 5        (installed + logged in, for live data)

Steps
-----
  1. Clone / pull the repository, e.g. to  C:\Mt5

  2. Build the dashboard once:
         cd C:\Mt5\frontend
         yarn install
         yarn build

  3. Install + start everything (one click):
         Right-click  C:\Mt5\installer\install.bat
         ->  "Run as administrator"

     That single command will:
        - auto-detect Python and the bundled NSSM (installer\nssm.exe)
        - install the required Python packages
        - register the AtlasBridge and AtlasBackend Windows services
        - start them and open  http://localhost:8001/

     Prefer to do it by hand? From an *elevated* command prompt:
         cd C:\Mt5\installer\scripts
         install_services.bat
         start_atlas.bat

Connect your MetaTrader 5 account
---------------------------------
  1. copy  C:\Mt5\mt5-bridge\.env.example  ->  C:\Mt5\mt5-bridge\.env
  2. fill in MT5_LOGIN / MT5_PASSWORD / MT5_SERVER and a BRIDGE_TOKEN
  3. add the same values to the backend so it can reach the bridge -
     create  C:\Mt5\backend\.env  with:
         MT5_BRIDGE_URL=http://127.0.0.1:8002
         MT5_BRIDGE_TOKEN=<the same BRIDGE_TOKEN>
  4. restart:  scripts\stop_atlas.bat  &&  scripts\start_atlas.bat

NSSM is missing?
----------------
A 64-bit nssm.exe is committed at  installer\nssm.exe  so a fresh clone
works offline. If it is ever removed, install_services.bat will try to
download NSSM 2.24 automatically; otherwise drop a copy at that path.

Useful URLs
-----------
  Dashboard : http://localhost:8001/
  Health    : http://localhost:8001/healthcheck
  API root  : http://localhost:8001/api/

Logs live under  <repo>\logs\  and data under  <repo>\data\.
